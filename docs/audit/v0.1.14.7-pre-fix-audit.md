# BiliLiveCut v0.1.14.7-alpha 修复前审计报告

**生成时间**: 2026-07-09  
**审计范围**: 整个仓库 (commit: 731a31c → HEAD)  
**审计目标**: 识别 Portable、Engine Pack、模型版本控制、原生扩展、Windows EXE、Full 离线包、完整性校验、运行时升级与发布流水线中的系统性问题

## 1. 版本号分散问题 — 已修复

**发现**:
- 版本号 `0.1.14.6-alpha` 在 20+ 处硬编码
- 位置: `app/__init__.py`, `pyproject.toml`, `setup.py`, `setup_c.py`, `manifest.py` (2处), `builder.py`, `main.py` (2处), `lite.py`, `full.py`, `engine_pack/builder.py`, `engine_pack/manifest.py`, `engine_pack/splitter.py`, `runtime_layout.py`, `model_downloader.py`, `__init__.py` (blc_portable), `build_rust.py`, `Cargo.toml`
- 无单源版本管理，修改版本需要逐一改动 15+ 个文件

**修复**:
- 创建 `packaging/portable/config/version.json` 作为唯一版本权威来源
- 创建 `version_loader.py` 提供统一加载接口
- 新增 `test_version_consistency.py` (12 项检查)
- 所有版本统一升级至 `0.1.14.7-alpha`

## 2. 模型定义重复 — 已修复

**发现**:
- `engine_pack/builder.py` 定义 `ENGINES` 常量 (4 个模型 + 子模型)
- `engine_pack/manifest.py` 定义另一份 `ENGINES` 常量 (与 builder 不一致!)
- `launcher/model_downloader.py` 硬编码模型 ID
- manifest.py 中 `funasr_nano` 的 model_id 为错误的 `iic/Fun-ASR-Nano` (此仓库在 ModelScope 不存在)
- manifest.py 中 SenseVoice 和 FunASR-Nano 被标记为 `revision: v2.0.4` (这些模型没有 v2.0.4 tag)
- manifest.py 中 Paraformer `model_id` 为缩写 `paraformer-zh` (应使用完整仓库 ID)
- manifest.py 中 Paraformer 子模型 cam++ 的 `model_id` 为短别名 `cam++` (应使用完整仓库 ID)
- manifest.py 中 SenseVoice `model_id` 为 `iic/SenseVoiceSmall` (格式一致但 revision 不匹配)

**修复**:
- 创建 `packaging/portable/config/model_sources.lock.json` 为唯一模型权威来源
- 创建 `model_catalog.py` 提供统一加载和验证接口
- builder.py 和 manifest.py 改为从 catalog 加载
- 修正 FunASR-Nano 为 `FunAudioLLM/Fun-ASR-Nano-2512`
- 增加 `resolved_revision` / `required_files` / `license` 等字段
- 新增 `test_model_catalog.py` (11 项检查)

## 3. CRC32 空校验问题 — 已修复

**发现**:
- `builders/lite.py` 的 `check_engine_pack_info()` 在 `engine_pack_info.json` 不存在时自动创建带空 CRC32 的占位
- `launcher/main.py` 也自动创建空 CRC32 的 pack_info 回退
- 空 CRC32 意味着 EXE 无法验证 Engine Pack 完整性
- 构建成功但功能不完整

**修复**:
- Lite builder 现在拒绝构建空 CRC32/SHA-256 的 EXE，抛出 RuntimeError
- Engine Pack installer 增加 SHA-256 强制校验 (安装条件)
- CRC32 仅作快速损坏检测
- 安装清单 schema 升级至 v2，增加 zip_sha256/source_commit 等字段

---

## 4. Runtime Release ID — 已修复

**发现**:
- `RELEASE_ID` 仅由 `version + source_commit` 组成 (`0.1.14.6-alpha+731a31c`)
- 不包含 Payload SHA-256，两个不同 Payload 可能使用相同 Release ID
- `get_current_release_dir()` 仅检查 `app/cli.py` 存在，不检查 Payload 是否一致
- 新 EXE 带不同 Payload 时可能复用旧 Runtime

**修复**:
- Release ID 改为内容寻址: `{version}+{source_commit}+{payload_hash_prefix}`
- `get_current_release_dir()` 增加 Payload SHA 比较
- `current.json` 增加 `runtime_schema`/`python_abi`/`platform`/`architecture`/`manifest_sha256` 字段

---

## 5. 非流式解压 — 已修复

**发现**:
- `launcher/main.py` 使用 `target.write_bytes(zf.read(member))` 将整个文件读入内存
- `engine_pack/installer.py` 使用 `dest.write_bytes(zf.read(member))`
- 不检查 totalextract_size、ZIP Bomb

**修复**:
- 改为 `shutil.copyfileobj(src, dst, length=8*1024*1024)` 流式解压
- Engine Pack installer 增加 20GB 解压上限
- 路径检查使用 `Path.resolve()` 和 `str.startswith()` 判断越界

---

## 6. Full 包实际内容 — 已修复

**发现**:
- `builders/full.py` 的 `build_full_bundle()` 仅复制 Lite EXE + README + checksums
- 不包含 portable-python、vendor/wheels、FFmpeg/FFprobe
- 无法在无 Python 环境的机器上运行

**修复**:
- 增加 portable-python/ 复制逻辑 (如果存在)
- 增加 vendor/wheels/ 复制 (如果存在)
- 增加 bin/ffmpeg.exe 和 bin/ffprobe.exe 复制 (如果存在)
- 组件缺失时输出明确警告 (不再静默构建无功能包)
