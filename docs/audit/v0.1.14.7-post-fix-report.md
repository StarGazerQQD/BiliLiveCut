## 3. CRC32 / SHA-256 校验链

```
Engine Pack ZIP
  ├── CRC32 (快速损坏检测, 8位十六进制)
  ├── SHA-256 (强制安装条件, 64位十六进制)
  └── 内部 Manifest SHA-256
       └── 引擎列表 + 逐文件 SHA-256

engine_pack_info.json (外部, 嵌入 EXE)
  ├── CRC32
  ├── SHA-256
  ├── Manifest SHA-256
  ├── 期望引擎 ID 列表
  └── 兼容 App 版本范围

Payload ZIP
  ├── SHA-256 (嵌入 Manifest)
  └── 逐文件 SHA-256 (内部 Manifest)

current.json (Runtime)
  ├── Payload SHA-256
  ├── Manifest SHA-256
  └── 平台/Python ABI 信息
```

---

## 4. Runtime 更新链

```
EXE 启动
  → 读取内嵌 Payload SHA-256
  → 读取 current.json 中的 installed Payload SHA-256
  → 比较:
     匹配 → 复用 Runtime
     不匹配 → 安装新 Runtime (内容寻址 Release ID)
  → 新 Release 安装:
     1. 创建 runtime/staging/
     2. 流式解压 Payload
     3. 校验 SHA-256
     4. 原子 rename 到 runtime/releases/{version}+{commit}+{payload_hash}/
     5. 原子写入 current.json
  → 保留旧 Release 作为回滚
```

## 8. 已知限制

1. **Full 离线包**: 需要预先准备 portable-python、wheels、FFmpeg。当前构建在缺失时会警告但继续（仅打包已存在内容）
2. **原生 ABI 分发包**: 未生成分 ABI 的 Payload (py311/py312)。所有构建在当前 Python 版本下进行
3. **AVX2 回退**: 未实现。当前仅支持 AVX2 架构（文档中注明最低 CPU 要求）
4. **代码签名**: 未配置 Authenticode 签名的接口
5. **Engine Pack 真实下载**: 需要 ModelScope/HuggingFace 访问权限
6. **跨平台**: macOS/Linux portable 未测试
